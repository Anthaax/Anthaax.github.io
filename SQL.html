Le SQL


Le SQL est un langage servant à exploiter les bases de données relationnelles.

Durant le projet ARCHI’VITE, pour stocker les données nous avons choisi d’utiliser SQL Server. Le but principal du semestre était d’utiliser un ORM (object-relational mapping) pour faciliter la communication entre la base de données et le code C#.
Nous avons choisi Entity Framework pour cela.

Bien que la base de données soit assez petite, certaines requêtes furent complexes car demandant plusieurs données à la fois dans différentes tables.

Grâce à ce projet j’ai appris à utiliser Entity Framework. Sur le moment j’ai apprécié utiliser ce Framework.
Mais avec le recul et les cours j’ai compris quelque chose de très important. Dans le développement il y a plusieurs frontières et les ORM ne respectent pas la frontière entre le serveur et la base de donnée et ne permet pas l'optimisation de la base de données. De plus, le modèle objet doit etre conçu de manière différente du modèle relationnel, ce que les ORM ne respectent pas. Pour des petits projets, les ORM sont très pratiques mais pour des projets de plus grande envergure la maintenance peut être difficile en raison du manque d'optimisation.

Le NoSQL


Le NoSQL est une famille de systèmes de gestion de base de données qui s’écarte du principe classique des bases de données relationnelles. Dans SENSE nous utilisons des bases de données NoSQL orienté-graphes (Cassandra, Neo4J).
J’ai apprécié l’approche de ce genre de base de données, très efficace sur un projet comme SENSE, qui utlise le principe de relation entre les diffrents signaux de l'expérimentation.

En revanche, sur certains volets, je le trouve moins efficace qu'une base de données SQL comme SQL Server, commme la facilité de retrouver de la données à partir de requête.
