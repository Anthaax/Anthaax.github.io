Le SQL


Le SQL est un langage servant à exploité les base de données relationnelles.

Durant le projet ARCHI’VITE pour stocker les données nous avons choisit d’utiliser SQL Server. Le but principal du semestre était d’utiliser un ORM pour faciliter la communication entre la base de données et le code C#.
Nous avons utilisé Entity Framework pour cela.

Bien que la base de données soit assez petite. Certaines requêtes furent assez grande car demandant plusieurs choses à la fois dans différentes tables.

Grace à ce projet j’ai appris à utiliser Entity Framework. Sur le moment j’ai apprécié utiliser ce Framework.
Mais avec le recul et les cours j’ai compris quelque chose de très important. Dans le développement il y a plusieurs frontières et les ORM ne respecte pas cette frontière qui est primordiale car un modèle objet n’est pas l’équivalent du modelé relationnel. Il faut donc penser le model objet de manière différentes que le modèle relationnel, mais malheureusement les ORM ne respecte pas cette consigne. Pour des petits projets les ORM sont très pratique mais pour des projets à plus grande envergure cela peut poser des problèmes pour la maintenance.

Le NoSQL


Le NoSQL est une famille de base systèmes de gestion de base de données qui s’écarte du principe classique des bases de données relationnelles. Dans SENSE nous utilisons des bases de données NoSQL orienté-graphes (Cassandra, Neo4J).
J’ai apprécié l’approche de genre de base de données qui est très efficace sur des projets comme SENSE.

Après sur certain point je les trouve extrêmement moins efficaces que des bases de données SQL normale comme SQL Server.
